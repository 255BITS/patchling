How it works

Three calls. No filesystem. No agent loop.

patchling is deliberately bounded: it transforms an in-memory file map and hands control straight back to you. You own the loop, the UI, and the storage.

1

buildEnvironment(files)

Serialize your { path: content } map into the environment string the model sees. No disk access — it runs in the browser and Node 18+.

2

generateDiff(env, goal)

One LLM call returns a unified git diff. Stream it live with onToken, meter it with onUsage, or inject your own client with callLlm.

3

smartapply(diff, files)

Deterministic patching first — context-matched, tolerant of drifted line numbers. Only files that still fail get a per-file LLM rewrite. Returns a new map; failures never partially write.

LLM diffs are sloppy. smartapply survives:

drifted @@ line numbers
context that doesn't quite match
malformed --/++ headers
headerless diffs
*** Begin Patch style
new files & deletions
multi-file diffs, applied concurrently
<think> & reasoning preambles

Get it

One primitive, two runtimes.

JavaScript

Zero-dependency ESM. Import from npm or straight off a CDN — this page does.

npm install patchling
# or, in the browser, no build step:
import { generateDiff, smartapply }
  from "https://esm.sh/patchling";
